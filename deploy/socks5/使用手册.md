# SOCKS5 代理使用手册

在新加坡服务器上独立运行的 SOCKS5 代理，供 **Telegram、浏览器、单个 App** 按需配置使用。

与 WireGuard **完全独立**，互不影响：

| 服务 | 协议 | 端口 | 用途 |
|------|------|------|------|
| WireGuard | UDP | 51820 | 系统级 VPN（WireGuard / Shadowrocket 全局或分流） |
| SOCKS5 | TCP | 1080 | 单个 App 内配置代理（如 Telegram） |

---

## 一、首次部署

### 从 Mac 一键安装

```bash
cd /Users/mac/Documents/own/www.ssyd.fun/telegram-feishu-forwarder
SSHPASS='Yzlh@123' ./deploy/setup-socks5-proxy.sh
```

终端会输出 **用户名** 和 **密码**，请保存好。

### 腾讯云防火墙

路径：**轻量应用服务器** → **tg-feishu-sg** → **防火墙**

| 协议 | 端口 | 来源 | 策略 |
|------|------|------|------|
| TCP | 1080 | 全部 IPv4 | 允许 |

> WireGuard 的 UDP 51820 规则保持不变，两者互不冲突。

## 二、Telegram 配置

1. 打开 Telegram → **设置** → **数据和存储** → **代理**
2. 点 **添加代理** → 选 **SOCKS5**
3. 填写：

| 字段 | 值 |
|------|-----|
| 服务器 | `43.160.239.73` |
| 端口 | `1080` |
| 用户名 | 部署时输出的用户名（默认 `proxy`） |
| 密码 | 部署时输出的密码 |

4. 保存并 **启用** 该代理

**所有设备可填同一套账号密码**，多设备同时在线无冲突。小火箭 / Clash 也支持添加 SOCKS5 节点。

### 查看已保存的凭证

```bash
ssh ubuntu@43.160.239.73 'sudo cat /root/socks5-credentials.txt'
```

## 三、其它 App / 客户端配置

### iOS / Android 支持 SOCKS5 的 App

在 App 的代理设置中选择 **SOCKS5**，填入相同参数即可。

### Mac 终端（临时使用）

```bash
export ALL_PROXY=socks5://proxy:你的密码@43.160.239.73:1080
curl https://ip.sb
```

### Mac 系统设置（仅部分 App 生效）

**系统设置** → **网络** → **Wi‑Fi** → **详细信息** → **代理** → 选手动代理，填 SOCKS 服务器信息。

> 系统级 SOCKS 会影响所有走系统代理的程序，若只想代理单个 App，优先在 App 内配置。

### Shadowrocket 中添加 SOCKS5 节点

1. **+** → 类型选手动输入或 **SOCKS5**
2. 填写服务器、端口、用户名、密码
3. 在 **配置 / 场景** 规则里，可指定只有 Telegram 域名走该节点，例如：

```
DOMAIN-SUFFIX,telegram.org,SOCKS5节点名
DOMAIN-SUFFIX,t.me,SOCKS5节点名
FINAL,DIRECT
```

这样 Shadowrocket 也可只对 TG 走 SOCKS5，其它流量直连，与 WireGuard 节点并存、按需切换。

## 四、与 WireGuard 并存说明

```
┌─────────────────────────────────────────────────┐
│  手机                                           │
│                                                 │
│  WireGuard App / Shadowrocket(WG节点)           │
│    → 系统级 VPN 或规则分流 → UDP 51820          │
│                                                 │
│  Telegram App（内置 SOCKS5 设置）               │
│    → 仅 TG 流量 → TCP 1080                      │
│                                                 │
│  微信 / 淘宝 / 其它 App                         │
│    → 直连，不受影响                              │
└─────────────────────────────────────────────────┘
```

- 可以同时安装 WireGuard 和配置 Telegram SOCKS5
- Telegram 优先使用 **App 内 SOCKS5 设置**（若已启用）
- 关闭 Telegram 代理后，TG 恢复直连
- WireGuard / Shadowrocket 开关与 SOCKS5 无关

## 七、常见问题

### 1. Telegram 代理连接失败

- 确认腾讯云防火墙已放行 **TCP 1080**
- 确认用户名密码正确
- 服务器检查：`sudo systemctl status microsocks`

### 2. 会影响 WireGuard 吗？

不会。两者端口、协议、服务进程完全独立。

### 3. 可以同时开 WireGuard 和在 TG 里开 SOCKS5 吗？

可以。通常 Telegram 会走 App 内配置的 SOCKS5；若 TG 未开代理，且 WireGuard 已连接，则 TG 走 WireGuard 隧道。

### 4. 只有 SOCKS5、不用 WireGuard 可以吗？

可以。只部署 SOCKS5，只在需要的 App 里配置即可。
