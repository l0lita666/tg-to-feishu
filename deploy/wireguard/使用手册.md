# WireGuard VPN 使用手册

新加坡服务器个人 VPN，供手机 / 电脑连接使用。

| 项目 | 值 |
|------|-----|
| 服务器 IP | `43.160.239.73` |
| 协议 / 端口 | UDP `51820` |
| VPN 内网段 | `10.8.0.0/24` |
| 服务端内网 IP | `10.8.0.1` |
| 客户端配置目录（服务器） | `/root/wireguard-clients/` |
| 服务端配置 | `/etc/wireguard/wg0.conf` |

---

## 一、客户端 App 选择

| App | 平台 | 说明 |
|-----|------|------|
| [WireGuard](https://www.wireguard.com/install/) | iOS / Android / Mac / Windows | 官方 App，稳定简单 |
| Shadowrocket（小火箭） | iOS | 支持 WireGuard，可配合规则分流 |

---

## 二、首次连接

### 方式 A：WireGuard 官方 App

1. 安装 WireGuard App
2. 点 **+** → **扫描二维码**（部署时终端输出的二维码）
3. 打开开关连接

### 方式 B：Shadowrocket（小火箭）

1. 打开 Shadowrocket → **+** → **扫描二维码**
2. 或手动新建，类型选 **WireGuard**，粘贴配置文件内容
3. 选中节点并连接

---

## 三、Shadowrocket 分流设置（推荐）

不想所有流量都走新加坡时，使用 **规则分流**：

### 全局路由选项说明

| 选项 | 含义 |
|------|------|
| **配置** | 按规则分流（国内直连，国外走代理）← **推荐** |
| **代理** | 全局走 VPN，所有流量经新加坡 |
| **直连** | 全局不走代理 |
| **场景** | 按预设场景切换策略，需事先配置场景 |

### 推荐设置

1. **设置** → **全局路由** → 选 **配置**（或已配置好的 **场景**）
2. 在 **配置** 中添加基础规则：

```
GEOIP,CN,DIRECT
FINAL,PROXY
```

3. 首页选中 WireGuard 节点并连接

### 分流效果验证

| 测试网站 | 预期结果 |
|----------|----------|
| Google、Telegram | 能正常打开 |
| [ip.sb](https://ip.sb) / [ifconfig.me](https://ifconfig.me) | 显示 `43.160.239.73`（新加坡） |
| [ip138.com](https://ip138.com) | 显示本地真实 IP（国内直连） |

## 四、添加新设备

每台设备需要独立配置（独立密钥），**不可多台同时共用同一配置**。

```bash
cd /Users/mac/Documents/own/www.ssyd.fun/telegram-feishu-forwarder

# 添加 iPad / Mac / 其它设备
SSHPASS='Yzlh@123' ./deploy/setup-wireguard-vpn.sh ipad
```

终端会输出新设备的 **二维码** 和 **配置文件**，用对应设备扫码导入。设备名自定义，避免重复。

### 在服务器上手动添加

```bash
ssh ubuntu@43.160.239.73
sudo bash /tmp/install-wireguard.sh 设备名称
```

若脚本不在 `/tmp`，可先上传：

```bash
# Mac 本地执行
scp deploy/wireguard/install-on-server.sh ubuntu@43.160.239.73:/tmp/install-wireguard.sh
# 服务器执行
sudo bash /tmp/install-wireguard.sh ipad
```

### IP 分配规则

| 设备 | 内网 IP |
|------|---------|
| 第一台（phone） | `10.8.0.2` |
| 第二台 | `10.8.0.3` |
| 第三台 | `10.8.0.4` |
| … | 自动递增 |

同一网段最多约 **253 台**（`10.8.0.2` ~ `10.8.0.254`），个人使用足够。

### 已添加的设备列表

```bash
ssh ubuntu@43.160.239.73 'sudo wg show wg0'
```

输出中每个 `peer` 即一台已注册设备；配置文件在：

```bash
ssh ubuntu@43.160.239.73 'sudo ls /root/wireguard-clients/'
```

## 六、删除设备

脚本暂未提供删除命令，需手动操作。

### 步骤

1. SSH 登录服务器：

```bash
ssh ubuntu@43.160.239.73
sudo nano /etc/wireguard/wg0.conf
```

2. 找到对应设备的 `[Peer]` 段（注释行 `# client: 设备名`），整段删除，例如：

```ini
# client: ipad
[Peer]
PublicKey = xxxxx
AllowedIPs = 10.8.0.3/32
```

3. 重载 WireGuard：

```bash
sudo wg syncconf wg0 <(sudo wg-quick strip wg0)
```

4. 删除客户端配置文件（可选）：

```bash
sudo rm /root/wireguard-clients/ipad.conf
```

5. 在手机上删除对应 VPN 配置。

## 九、常见问题

### 1. 连接不上 / 一直「连接中」

- 确认腾讯云防火墙已放行 **UDP 51820**
- 确认手机网络正常（可换 4G / WiFi 试）
- 服务器检查服务是否运行：

```bash
ssh ubuntu@43.160.239.73 'sudo systemctl status wg-quick@wg0'
```

### 2. 能连上但没有网

```bash
# 检查 IP 转发是否开启
ssh ubuntu@43.160.239.73 'cat /proc/sys/net/ipv4/ip_forward'
# 应输出 1
```

若为 `0`：

```bash
ssh ubuntu@43.160.239.73 'sudo sysctl -w net.ipv4.ip_forward=1'
sudo systemctl restart wg-quick@wg0
```

### 3. Google 能开，国内网站也变慢

Shadowrocket 全局路由可能设成了 **代理**，改为 **配置** 或 **场景**，并确认有 `GEOIP,CN,DIRECT` 规则。

### 4. 添加设备时提示「客户端已存在」

换一个新名称，例如 `phone2`、`ipad2`：

```bash
SSHPASS='Yzlh@123' ./deploy/setup-wireguard-vpn.sh phone2
```

### 5. SSH 连接超时

使用腾讯云控制台 → **在线终端** 登录，在服务器上直接执行安装脚本。

## 十、性能与设备数量

| 情况 | 影响 |
|------|------|
| 设备挂着不用 | 几乎无影响 |
| 多台同时看视频 / 下载 | 共享服务器带宽，可能变慢 |
| 个人 3～5 台 | 通常够用 |

瓶颈主要在 **腾讯云轻量机带宽**，而非 WireGuard 本身。不建议把 VPN 共享给大量用户使用。

---

## 十一、安全提醒

1. **PrivateKey 是私钥**，不要发给他人或发到公开渠道
2. 每台设备独立密钥，丢手机后可单独删除该设备的 Peer
3. 若私钥泄露，删除旧设备后重新 `./deploy/setup-wireguard-vpn.sh 设备名` 生成新配置
4. VPN 与 Telegram 转发服务共用一台服务器，互不影响，但共享带宽
