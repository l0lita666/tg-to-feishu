## 一、项目概述

本程序使用 **Telethon** 登录一个 Telegram 账号，监听指定 TG 群聊与全部私聊，将消息转发到飞书/Lark；在启用双向模式后，可在飞书侧发消息回到 TG。

**架构示意：**

```
TG 群 / 私聊
    │  Telethon 监听
    ▼
telegram_listener.py
    ├── TG → 飞书：Bot 卡片（文字 / 图片 / 相册 / 头像）
    └── 飞书 → TG：HTTP 事件回调（:8081/feishu/event）
            │
            ▼
        飞书群（一群对应一群 + 私聊汇总群）
```

**当前方案：**

| 类型 | 策略 |
|------|------|
| TG 群 | 一群对应一个飞书群（`FEISHU_CHAT_MAP`） |
| TG 私聊 | 汇总到一个飞书群（`FEISHU_DM_CHAT_ID`） |
| 飞书 → TG 身份 | 始终为登录的 TG 账号发出 |

## 二、完整功能清单

### 2.1 TG → 飞书

| 功能 | 说明 |
|------|------|
| 群聊转发 | `GROUP_MODE=manual` 时监听 `GROUP_CHATS` 中的群 |
| 私聊转发 | 所有私聊自动转发到私聊汇总群 |
| 文字消息 | 紧凑卡片：时间 · 群名 · 发送者 + 正文 |
| 用户头像 | 卡片左侧展示 TG 用户头像（有头像时；按 user_id 缓存） |
| 图片 / 贴纸 | 需配置 `FEISHU_APP_ID` + `FEISHU_APP_SECRET` |
| 相册合并 | 多张图合并为一条卡片转发 |
| 仅他人消息 | `INCOMING_ONLY=true` 时不转发自己发的 TG 消息 |

### 2.2 飞书 → TG

| 功能 | 说明 |
|------|------|
| 群聊直发 | 在对应飞书群直接发文字 / 图片，回到 TG 群 |
| 私聊汇总 | 须 **回复** 某人的转发卡片，才回到对应 TG 私聊（普通新消息，不引用） |
| 回复卡片自动 @ | 飞书 **回复 TG 转发卡片** 时，自动 `@` 原 TG 发言者并 `reply_to` 原消息 |
| 飞书 @ 占位符过滤 | 自动去掉飞书内部的 `@_user_1` 等占位符 |
| 图片转发 | 飞书发图 → 下载后发到 TG |
| 去重 | 同一飞书 `message_id` 只处理一次，避免 Lark 重试导致重复发送 |
| 成功无提示 | 发送成功不弹机器人提示；失败才提示 |
| 操作者限制 | `FEISHU_ALLOWED_OPEN_IDS` 留空 = 飞书群内所有人可操作 |

### 2.3 运维

| 功能 | 说明 |
|------|------|
| 断线重连 | 自动重连 TG，间隔 `RECONNECT_INTERVAL` 秒 |
| Ctrl+C 退出 | 前台模式可正常退出 |
| 后台运行 | `./start.sh start` + systemd |
| 消息映射 | SQLite 存飞书卡片 ↔ TG 消息，供回复与 @ 使用 |

## 三、配置项说明（`.env`）

复制 `config.example.env` 为 `.env` 后填写。

### Telegram

| 变量 | 说明 |
|------|------|
| `TELEGRAM_API_ID` / `TELEGRAM_API_HASH` | 从 https://my.telegram.org 获取 |
| `TELEGRAM_PHONE` | 登录手机号（含国际区号） |
| `SESSION_NAME` | session 文件名，默认 `telegram_session` |
| `TELEGRAM_PROXY` | 大陆服务器 **必填** 代理；Mac 本地一般留空 |
| `GROUP_MODE` | `manual` 手动指定群 / `unmuted` 所有未静音群 |
| `GROUP_CHATS` | manual 模式下监听的 TG 群 ID，逗号分隔 |
| `INCOMING_ONLY` | `true` 只转发他人消息 |

### 飞书

| 变量 | 说明 |
|------|------|
| `FEISHU_WEBHOOK_URL` | 自定义机器人 Webhook（降级备用） |
| `FEISHU_APP_ID` / `FEISHU_APP_SECRET` | 开放平台应用凭证 |
| `FEISHU_REPLY_ENABLED` | `true` 启用双向 |
| `FEISHU_CHAT_MAP` | `TG群ID:飞书chat_id`，多个逗号分隔 |
| `FEISHU_DM_CHAT_ID` | 私聊汇总飞书群 chat_id |
| `FEISHU_EVENT_PORT` | 事件 HTTP 端口（默认 8080；与占用端口冲突时可改 8081） |
| `FEISHU_EVENT_PATH` | 事件路径，默认 `/feishu/event` |
| `FEISHU_VERIFICATION_TOKEN` | 开放平台 → 事件与回调 → **加密策略** |
| `FEISHU_ENCRYPT_KEY` | 启用加密时填写 |
| `FEISHU_ALLOWED_OPEN_IDS` | 允许操作的飞书用户 open_id，留空不限制 |

### 6.4 Nginx 反向代理 + HTTPS

```nginx
# /etc/nginx/sites-available/99ka.life
server {
    listen 443 ssl http2;
    server_name 99ka.life;

    ssl_certificate     /etc/letsencrypt/live/99ka.life/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/99ka.life/privkey.pem;

    # 飞书事件回调
    location /feishu/event {
        proxy_pass http://127.0.0.1:8081/feishu/event;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 60s;
    }

    # 可选：健康检查
    location /feishu/health {
        proxy_pass http://127.0.0.1:8081/health;
    }
}

server {
    listen 80;
    server_name 99ka.life;
    return 301 https://$host$request_uri;
}
```

## 七、新增 / 维护群聊

### 7.1 新增一个 TG 群 ↔ 飞书群

**步骤：**

1. **TG 侧**：确认桥接账号已加入该 TG 群  
2. **查 TG 群 ID**：
   ```bash
   ./start.sh groups
   ```
3. **飞书侧**：新建飞书群，拉入「文件转发」机器人  
4. **查飞书 chat_id**：
   ```bash
   ./start.sh feishu-chats
   ```
5. **改 `.env`**：
   ```env
   GROUP_CHATS=-5289292369,-5005244212,-新TG群ID
   FEISHU_CHAT_MAP=-5289292369:oc_xxx,-5005244212:oc_yyy,-新TG群ID:oc_新飞书群id
   ```
6. **重启**：
   ```bash
   ./start.sh restart
   # 或 sudo systemctl restart telegram-feishu
   ```
7. **测试**：TG 群发消息 → 飞书收到；飞书回复 → TG 收到

### 7.2 删除群映射

- 从 `GROUP_CHATS` 和 `FEISHU_CHAT_MAP` 中移除对应项
- 重启服务

### 7.3 修改群映射（换飞书群）

- 只改 `FEISHU_CHAT_MAP` 中对应的 `oc_xxx`
- 重启服务  
- **注意**：旧卡片上的回复映射可能失效，新转发的消息才有新映射

### 7.4 私聊汇总群

- 所有 TG 私聊进 `FEISHU_DM_CHAT_ID` 指定的飞书群
- 飞书 → TG 私聊：**必须回复** 转发卡片
- 换汇总群：改 `FEISHU_DM_CHAT_ID` 并重启

### 7.5 限制飞书操作者

只允许指定同事通过飞书发 TG：

```env
FEISHU_ALLOWED_OPEN_IDS=ou_xxx,ou_yyy
```

## 八、飞书 → TG 使用说明（给同事）

### 群聊

| 操作 | 效果 |
|------|------|
| 直接发文字 | 发到对应 TG 群 |
| 直接发图片 | 发到对应 TG 群 |

| 操作 | 效果 |
|------|------|
| 直接发消息 | ❌ 不会到 TG（会提示须回复卡片） |

在飞书消息中写 `@TG用户名`（对方须有 TG Username）。  
回复卡片时会自动 @，一般无需手打。

## 九、常见问题

| 现象 | 排查 |
|------|------|
| `address already in use` | 改 `FEISHU_EVENT_PORT`（如 8081），Nginx/ngrok 同步改端口 |
| 飞书收不到 TG 消息 | 查 `GROUP_CHATS`、群映射、日志「跳过：不在监听范围」 |
| 飞书 → TG 无反应 | 查 Lark 事件 URL、是否发布、`FEISHU_REPLY_ENABLED=true`、进程是否运行 |
| URL 验证失败 | 先启动 listener，再保存 Lark 地址；确认 HTTPS 可达 |
| 私聊转发失败 | 配置 `FEISHU_DM_CHAT_ID` |
| 图片 / 头像不显示 | 检查 `FEISHU_APP_ID/SECRET`、`im:resource` 权限 |
| 飞书发图 TG 收到两次 | 已做 message_id 去重；确认只跑一个实例 |
| 回复卡片无法 @ | 须回复 **重启后新转发** 的卡片（旧卡片无映射） |
| 大陆服务器连不上 TG | 配置 `TELEGRAM_PROXY` |
| session 失效 | 删除 session 文件，重新 `./start.sh login` |
